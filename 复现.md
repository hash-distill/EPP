# EPP（Epitope-Paratope Predictor）论文实验复现全流程详解
本文档基于原仓库代码与文档的偏差修正，详细梳理EPP模型（表位-对位预测器）从环境搭建、数据处理、模型训练到推理预测的完整复现流程，并补充核心原理与关键细节说明，确保实验可复现、可验证。

## 一、实验背景与核心原理
EPP模型旨在预测抗原（Antigen）与抗体（Antibody）之间的交互残基位点（表位/对位），核心流程为：
1. 利用ESM-2预训练模型提取氨基酸序列的高维特征表征；
2. 构建基于真实交互位点的监督标签矩阵；
3. 通过Bi-LSTM网络学习特征与交互位点的关联规律；
4. 基于训练后的模型对新序列对进行交互位点预测。

模型核心依赖**ESM-2的残基级特征提取**和**Bi-LSTM对序列上下文的建模能力**，最终通过特征矩阵相乘得到残基对的结合概率，实现二分类判别（结合/不结合）。

## 三、数据预处理（核心步骤）
数据预处理分为**ESM-2特征提取**和**监督标签矩阵构建**两部分，最终产出模型训练所需的3个核心张量文件。

### 3.1 原始数据结构说明
原始数据存储在`data/`目录下，包含两个核心文件：
| 文件路径       | 列数 | 列含义                                                                 |
|----------------|------|------------------------------------------------------------------------|
| `data/train.csv` | 5列  | 第0列：蛋白复合物名称/PDB ID；第1列：抗体标识、抗体氨基酸序列；第2列：抗原标识、抗原氨基酸序列 |
| `data/label.csv` | 3列  | 第0列：复合物名称/PDB ID；第1列：抗原长度(aglen)；第2列：抗体长度(ablen)；第3列：交互残基坐标对列表（如[[12,45],[13,21]]） |

### 3.2 ESM-2特征提取
#### 3.2.1 提取逻辑
1. 加载预训练模型`esm2_t33_650M_UR50D`；
2. 分别对抗体、抗原的氨基酸序列进行特征提取，取模型第33层的`representations`作为残基特征；
3. 对不同长度的序列进行zero-padding（零填充），统一特征矩阵维度；
4. 将特征矩阵保存为PyTorch张量文件，供训练脚本调用。

#### 3.2.2 执行命令
通过`nbconvert`静默执行Jupyter Notebook脚本：
```bash
# 提取抗原/抗体的ESM-2特征，生成traindata_esm_ag.pt和traindata_esm_ab.pt
EPP_TRAIN_CSV=data/custom/train.csv \
EPP_OUTPUT_DIR=data/custom \
jupyter nbconvert --to notebook --execute data/esm_representation.ipynb \
	--inplace --ExecutePreprocessor.timeout=-1
```

#### 3.2.3 输出文件说明
| 输出文件路径               | 含义                                                                 |
|----------------------------|----------------------------------------------------------------------|
| `data/custom/traindata_esm_ag.pt` | 抗原序列的ESM-2特征张量，维度为`[样本数, 最大序列长度, 特征维度]`     |
| `data/custom/traindata_esm_ab.pt` | 抗体序列的ESM-2特征张量，维度为`[样本数, 最大序列长度, 特征维度]`     |

### 3.3 监督标签矩阵构建
#### 3.3.1 标签构建逻辑
1. 读取`data/label.csv`中的交互残基坐标对；
2. 为每个样本构建`抗原长度×抗体长度`的二值矩阵：交互残基对位置设为1，其余为0；
3. 对矩阵进行zero-padding，统一维度后打包为三维张量（样本数×统一长度×统一长度）；
4. 保存为标签张量文件，注意训练脚本默认调用`tensor_label_y.pt`，需重命名或修改代码适配。

#### 3.3.2 执行命令
```bash
# 构建标签矩阵，生成label_y.pt
jupyter nbconvert --to notebook --execute data/label-y.ipynb
```

#### 3.3.3 输出文件说明
| 输出文件路径         | 含义                                                                 |
|----------------------|----------------------------------------------------------------------|
| `data/tensor_label_y.pt` | 监督标签张量，维度为`[样本数, 统一抗原长度, 统一抗体长度]`，值为0/1（不结合/结合） |

## 四、模型训练
### 4.1 训练核心逻辑
1. 加载预处理后的3个张量（抗原特征、抗体特征、标签矩阵）；
2. 配置Bi-LSTM超参数（如`hidden_size=256`、`learning_rate=1e-3`、`batch_size=32`等）；
3. 分别构建抗原、抗体分支的Bi-LSTM网络，对特征进行时序编码；
4. 将两个分支的编码结果进行矩阵乘法，得到残基对的结合概率矩阵；
5. 以二元交叉熵为损失函数，优化网络参数；
6. 通过阈值过滤（默认0.5）优化预测精度，保存训练后的模型权重。

### 4.2 执行训练
直接运行训练脚本`train.py`（需确保张量文件路径正确）：
```bash
python train.py --device cuda:1 \
	--ag-path data/custom/traindata_esm_ag.pt \
	--ab-path data/custom/traindata_esm_ab.pt \
	--label-path data/custom/label_y.pt
```

### 4.3 训练输出说明
| 输出文件路径          | 含义                                                                 |
|-----------------------|----------------------------------------------------------------------|
| `model/model_ag.pth`  | 抗原分支Bi-LSTM的模型权重，包含网络结构和训练后的参数                 |
| `model/model_ab.pth`  | 抗体分支Bi-LSTM的模型权重，包含网络结构和训练后的参数                 |

## 五、推理预测
### 5.1 预测脚本使用说明
调用`predict.py`可实现端到端的交互位点预测，输入为抗原和抗体的氨基酸序列，输出为结合残基的位置（表位/对位）。

#### 5.1.1 执行命令
```bash
python predict.py --agseq <抗原氨基酸序列> --abseq <抗体氨基酸序列>
```

#### 5.1.2 参数说明
| 参数    | 含义                                                                 | 输入示例                                  |
|---------|----------------------------------------------------------------------|-------------------------------------------|
| --agseq | 抗原氨基酸序列（如病毒Spike蛋白片段）| TNLCPFHEVFNATTFASVYAWNRTRISNCVADYSVLYNFAPFFAFKCYGVSPTKLNDLCFTNVYADSFVIKGNEVSQIAPGQTGNIADYNYKLPDDFTGCVIAWNSNKLDSRRSGNYDYWYRSLRKSKLKPFERDISTEIYQAGNKPCKGKGPDCYFPLESYGFRPTYGVGHQPYRVVVLSFELLHAPATVCGP                |
| --abseq | 抗体氨基酸序列（重链可变区）| QSVEESGGRLVTPGTPLTLTCTVSGFSLNSYDMGWFRQAPGKGLEYIGMIGLSGTYYASWAKGRFTFSYTSTTMELKIASPTTEDAATYFCAREDLLRFDLWGQGTLVTVSS               |

### 5.2 预测核心流程
1. 加载训练好的`model_ag.pth`和`model_ab.pth`；
2. 对输入的新序列提取ESM-2特征（复用预处理阶段的特征提取逻辑）；
3. 将特征输入Bi-LSTM网络，得到编码后的特征矩阵；
4. 特征矩阵相乘得到残基对结合概率矩阵；
5. 以`threshold=0.5`为判别标准，筛选出概率≥0.5的残基对；
6. 输出表位（抗原侧结合残基位置）和对位（抗体侧结合残基位置）。

## 六、关键注意事项
1. **文件路径一致性**：训练脚本默认读取`data/tensor_label_y.pt`，需确保标签文件命名正确；
2. **序列长度适配**：ESM-2特征提取和标签构建阶段的padding长度需保持一致，否则会导致维度不匹配；
3. **超参数调整**：训练时可根据数据规模调整`batch_size`、`learning_rate`、`threshold`等参数，优化预测精度；
4. **环境依赖**：确保全程在`EPP` conda环境下执行，避免ESM-2模型加载失败或依赖缺失；
5. **硬件要求**：ESM-2模型（650M参数）和Bi-LSTM训练需至少8GB显存，建议使用GPU加速。

## 七、问题排查指引
1. **Notebook执行失败**：检查`nbconvert`是否安装，或Notebook中ESM-2模型路径是否正确；
2. **张量维度不匹配**：核对特征张量和标签张量的padding长度，确保样本数、序列长度一致；
3. **预测结果全为0/1**：调整`threshold`阈值，或重新训练模型（可能是学习率过高/过低导致欠拟合/过拟合）；
4. **模型加载失败**：检查`model/`目录下的权重文件是否存在，或训练时是否正常保存参数。

通过以上流程，可完整复现EPP模型的训练与预测过程，若需验证模型效果，可使用已知交互位点的序列对进行预测，对比真实位点与预测位点的重合度。
